    cmake: è un sistema di build cross-platform che genera file di progetto per diversi ambienti di sviluppo. Utilizza un file di configurazione chiamato CMakeLists.txt per definire le regole di compilazione e le dipendenze. CMake supporta una vasta gamma di linguaggi di programmazione e può essere utilizzato per progetti di qualsiasi dimensione.
    bazel: è un sistema di build sviluppato da Google che si concentra sulla velocità e l'efficienza. Utilizza un file di configurazione chiamato BUILD per definire le regole di compilazione e le dipendenze. Bazel supporta una vasta gamma di linguaggi di programmazione e può essere utilizzato per progetti di qualsiasi dimensione.
    meson: è un sistema di build moderno che si concentra sulla facilità d'uso e sulla velocità. Utilizza un file di configurazione chiamato meson.build per definire le regole di compilazione e le dipendenze. Meson supporta una vasta gamma di linguaggi di programmazione e può essere utilizzato per progetti di qualsiasi dimensione.
    ninja: è un sistema di build leggero e veloce che si concentra sulla velocità di compilazione. Utilizza un file di configurazione chiamato build.ninja per definire le regole di compilazione e le dipendenze. Ninja è progettato per essere utilizzato come backend per altri sistemi di build, come CMake e Meson.
    scons: è un sistema di build scritto in Python che utilizza un file di configurazione chiamato SConstruct per definire le regole di compilazione e le dipendenze. Scons supporta una vasta gamma di linguaggi di programmazione e può essere utilizzato per progetti di qualsiasi dimensione.
    gradle: è un sistema di build basato su Java che utilizza un file di configurazione chiamato build.gradle per definire le regole di compilazione e le dipendenze. Gradle è progettato per essere utilizzato con progetti Java, ma supporta anche altri linguaggi di programmazione.
    maven: è un sistema di build basato su Java che utilizza un file di configurazione chiamato pom.xml per definire le regole di compilazione e le dipendenze. Maven è progettato per essere utilizzato con progetti Java e gestisce le dipendenze in modo efficiente.
    ant: è un sistema di build basato su Java che utilizza un file di configurazione chiamato build.xml per definire le regole di compilazione e le dipendenze. Ant è progettato per essere utilizzato con progetti Java, ma supporta anche altri linguaggi di programmazione. Ant è noto per la sua flessibilità e la capacità di gestire progetti complessi.


Pipeline
 latenza e trhoughput
    latenza: tempo necessario per completare un'operazione
    throughput: numero di operazioni completate in un dato intervallo di tempo
    in una pipeline, la latenza è il tempo necessario per completare un'operazione dall'inizio alla fine, mentre il throughput è il numero di operazioni completate in un dato intervallo di tempo


esecuzione out of order e instruction level parrallelism
    anche se posso eseguiro n istruzioni in parallelo duplicando hardware non è detto che n istruzioni onscutive siano sempre eseguibili in parallelo
    possiamo cambiare l'ordi e di eseccuzioni delle istruzioni

ciclo fetch decode execute
    fetch: recupera l'istruzione dalla memoria
    decode: decodifica l'istruzione per capire cosa deve fare
    dispatch istruzione viene isnerita
    execute: esegue l'istruzione

bolla nel esecuzione
    se un'istruzione dipende da un'altra istruzione che non è ancora stata completata, si crea una bolla nella pipeline, che è un ciclo in cui nessuna istruzione viene eseguita. Questo può ridurre il throughput della pipeline e aumentare la latenza delle operazioni.

structural hazard: due istruzioni necessitano della stessa parte della cpu
data hazard: un istruzione necessita dati da un operando che deve essere ancora computato
control hazard: non posso eseguire nessuna altra istruzione fino a quando non sappiamo la prossima istruzione da eseguire
